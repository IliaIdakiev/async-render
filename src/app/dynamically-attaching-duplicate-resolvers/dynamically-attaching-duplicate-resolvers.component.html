5. Dynamically Attaching Duplicate Resolvers (with unique ids)

You can use this for attaching directives to the resolve container dynamically (when using ngIf / ngSwitch / ngFor). In the current case we don't need to duplicate the userPostsResolver because we can just reuse the template reference variavble into multiple places but this is very useful if you have multiple places where you use the same resolver and they can be conditionally visible. In that case we can't be sure which resolver will pre visible so the first one visible will be the main one and all the others (that might get visible later) will be connected to the main one.
